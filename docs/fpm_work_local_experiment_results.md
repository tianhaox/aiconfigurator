# FPM Work-Local Experiment Results

This document records the static-ground-truth experiment results behind the
FPM work-local methodology. It intentionally does not include generated CSV
data.

## Setup

Ground truth source:

```text
static AIC latency CSVs generated for prefill/decode FPM shape studies
```

Compared predictors:

- `raw_idw`: direct shape-space log-latency IDW baseline.
- `work_local`: local affine model in forward-pass work feature space.

Primary metrics:

```text
MAPE, P95 absolute percentage error, P99 absolute percentage error, Max APE
```

The numbers below are from validation/test splits that are not part of the
training grid.

## Collection-Latency Estimate

Pure collection latency is computed as:

```text
sum(latency_ms over train shapes)
```

This excludes model load, collector startup, warmup, repeats, and Python
overhead. Real collection wall time should multiply the per-point latency by
the configured repeat count and add collector overhead.

| Model | old 2051 | old 1908 prefix-sparse | old 1809 | new 773 |
| --- | ---: | ---: | ---: | ---: |
| Qwen3-235B | 26.83 min | 17.81 min | 23.65 min | 7.20 min |
| Qwen3-32B | 13.08 min | 8.69 min | 11.52 min | 3.53 min |
| DeepSeek-V3 | 46.96 min | 32.08 min | 41.14 min | 12.84 min |
| Llama3.1-8B | 20.89 min | 14.27 min | 18.31 min | 5.85 min |

Reduction from old compressed shape-space plans to the 773-point work-local
plan:

```text
vs old 2051: about 72%-73%
vs old 1809: about 68%-70%
vs old 1908 prefix-sparse: about 59%-60%
```

## Interpretation

The work-local model replaces direct shape-space interpolation as the main
candidate. The shape-space baseline remains useful as a debug baseline or
fallback, but it needs more points to reach similar tail behavior.

The key empirical observations are:

- Global affine work modeling is not enough.
- Local affine work modeling improves both MAPE and tail error.
- Prefix coverage should not be compressed too aggressively.
- Batch and sequence anchors can be compressed substantially when work features
  are used.
- Very short prefill and decode batch/KV boundary regions remain the main
  regimes that need careful local-neighborhood tuning.
